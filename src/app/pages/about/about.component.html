Purpose

On November 8th, 2016, Donald Trump was elected to be the 45th President of the United States of America. Since then, President Trump has tweeted over ~2,500 times using his @realDonaldTrump Twitter account.

We sought out to answer a simple question:
"How do the people feel about President Trumps tweets?"

In order to answer this question, we needed data, lots of it. What better place to turn then Twitter where President Trump makes public statements dozens of times per day?

This project was created for the CSE 482 course at Michigan State University.

What we do

Every time President Trump posts a tweet, our program does the following:


  1. Store the Tweet in our database.
  2. Track the replies of the tweet.
  3. Analyze the sentiment of each individual reply. Sentiment can be positive, negative, or neutral.
  4. Count the frequency of each hashtags used in each reply.
  5. Visualize the data collected on this website.

The Team

Technologies we used

  • Angular

    This website that you're using to view our analysis is built from the ground up using the Angular web framework.

  • Google Firebase

    The website you're viewing right now was sent to you from Firebases' hosting servers!

    The data you see on the homepage is all saved on Firebase as well.

    By utilizing Firebase Functions we were able to handle the processing of the replies with a completely serverless architecture.

  • Node Sentiment Library

    We used the AFINN-based sentiment analysis library for Node.js to analyze the text in the replies.

  • Travis CI

    Travis CI helped us automate our build and deployment process so we could stress less about the logistics of getting our application up and running and more on actually making the application.

    Thanks for all the help, Travis!

  • Tweepy

    We used Tweepy to collect all of the Tweets and replies from Twitter.
